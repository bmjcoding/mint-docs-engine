The Users API lets you manage user accounts programmatically. All endpoints require authentication via API key or OAuth token.

## List Users

Retrieve a paginated list of users in your project.

```bash
GET /v1/users
```

### Query Parameters

| Parameter | Type | Description |
|-----------|------|-------------|
| `limit` | integer | Maximum number of results (1-100, default 20) |
| `offset` | integer | Number of results to skip |
| `email` | string | Filter by email address |
| `created_after` | string | ISO 8601 timestamp filter |

### Response

```json
{
  "data": [
    {
      "id": "usr_abc123",
      "email": "jane@example.com",
      "name": "Jane Doe",
      "role": "admin",
      "created_at": "2026-03-21T14:30:00Z",
      "updated_at": "2026-03-21T14:30:00Z"
    }
  ],
  "has_more": true,
  "total": 142
}
```

## Create User

Create a new user in your project.

```bash
POST /v1/users
```

### Request Body

```json
{
  "email": "jane@example.com",
  "name": "Jane Doe",
  "role": "member",
  "metadata": {
    "department": "Engineering"
  }
}
```

<Note>
  The `email` field must be unique within your project. Attempting to create a user with a duplicate email will return a `409 Conflict` error.
</Note>

### Response

```json
{
  "id": "usr_abc123",
  "email": "jane@example.com",
  "name": "Jane Doe",
  "role": "member",
  "metadata": {
    "department": "Engineering"
  },
  "created_at": "2026-03-21T14:30:00Z",
  "updated_at": "2026-03-21T14:30:00Z"
}
```

## Get User

Retrieve a single user by ID.

```bash
GET /v1/users/:id
```

<Tabs>
  <Tab title="cURL">
    ```bash
    curl -X GET https://api.acme.com/v1/users/usr_abc123 \
      -H "Authorization: Bearer sk_live_abc123"
    ```
  </Tab>
  <Tab title="Node.js">

  </Tab>

## Update User

Update an existing user's properties.

```bash
PATCH /v1/users/:id
```

### Request Body

```json
{
  "name": "Jane Smith",
  "role": "admin",
  "metadata": {
    "department": "Leadership"
  }
}
```

<Tip>
  Only include the fields you want to update. Omitted fields will remain unchanged.
</Tip>

## Delete User

Permanently delete a user and all associated data.

```bash
DELETE /v1/users/:id
```

<Danger>
  This action is irreversible. All user data, including activity history and associated resources, will be permanently deleted.
</Danger>

### Response

```json
{
  "id": "usr_abc123",
  "deleted": true
}
```

## Error Codes

| Status | Code | Description |
|--------|------|-------------|
| 400 | `invalid_request` | The request body is malformed |
| 401 | `unauthorized` | Missing or invalid API key |
| 404 | `not_found` | User does not exist |
| 409 | `conflict` | Email already in use |
| 429 | `rate_limited` | Too many requests |
